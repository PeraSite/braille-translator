# Repository Instructions

## Commit Messages

Use Conventional Commits with Korean descriptions.

Format:

```text
type: 변경 내용 요약
```

Examples:

```text
feat: 한국어 점역 기능 추가
fix: 영어 특수 문자 오류 대응 개선
refactor: 점자 변환 모듈 구조 분리
docs: 커밋 메시지 규칙 추가
```

Common types:

- `feat`: 기능 추가
- `fix`: 버그 수정
- `refactor`: 동작 변경 없는 구조 개선
- `docs`: 문서 변경
- `test`: 테스트 추가 또는 수정
- `chore`: 설정, 빌드, 기타 유지보수
